## 1. 개요

PointNGD는 타겟 프로그램(게임 창)에서 특정 UI 요소를 인식하기 위해 **템플릿 매칭(Template Matching)** 방식을 사용합니다. 이를 위해 두 가지를 사전에 설정해야 합니다:

- **ROI (Region of Interest)**: 화면에서 특정 UI 요소를 찾을 **검색 영역**의 좌표와 크기
- **템플릿 이미지**: 해당 영역에서 찾을 **기준 이미지** (PNG 파일)

이 설정은 `ROI & Image Editor`를 통해 시각적으로 수행할 수 있습니다.

---

## 2. 사전 준비

### 2.1 타겟 윈도우 타이틀 설정

설정 탭에서 **Target Window Title** 항목에 타겟 프로그램의 윈도우 제목(또는 일부)을 입력합니다.

> 예: `Lineage` 를 입력하면, 제목에 "Lineage"가 포함된 창을 자동으로 찾습니다.

### 2.2 타겟 프로그램 실행

ROI 설정을 시작하기 전에, 타겟 프로그램을 실행하고 설정하려는 화면 상태를 미리 준비해둡니다.

---

## 3. ROI & Image Editor 열기

에디터를 여는 방법은 두 가지입니다:

- **메뉴바**: `Tools` → `ROI & Image Editor`
- **설정 탭**: `Open ROI & Image Editor` 버튼 클릭

에디터가 열리면 자동으로 설정된 타겟 윈도우를 검색하며, 상단에 연결 상태가 표시됩니다:
- **녹색 "Found"**: 타겟 윈도우를 성공적으로 찾음
- **빨간색 "NOT FOUND"**: 타겟 윈도우를 찾을 수 없음 (프로그램 실행 여부 확인)

---

## 4. 타겟 윈도우 캡쳐하기

### 4.1 자동 캡쳐 (Capture Screen)

1. 에디터 하단의 **"Capture Screen"** 버튼을 클릭합니다.
2. 에디터가 잠시 숨겨진 후, 타겟 윈도우의 스크린샷이 촬영됩니다.
3. 캡쳐된 이미지가 오른쪽 캔버스에 표시됩니다.
4. 기존에 설정된 ROI 영역들이 색상 사각형으로 함께 표시됩니다.

> **참고**: Windows에서는 `PrintWindow()` API를 사용하여 다른 창에 가려져 있어도 정확하게 캡쳐됩니다. Linux에서는 `mss` 라이브러리를 사용합니다.

### 4.2 특정 윈도우 선택 캡쳐 (Capture Window...)

다른 윈도우를 캡쳐하고 싶은 경우:

1. **"Capture Window..."** 버튼을 클릭합니다.
2. 현재 열린 모든 윈도우 목록이 나타납니다.
3. 원하는 윈도우를 선택하면 해당 윈도우가 캡쳐됩니다.

---

## 5. ROI 영역 설정하기

### 5.1 드래그로 ROI 설정 (권장)

1. **왼쪽 목록에서 설정할 ROI 항목을 클릭**하여 선택합니다.
2. **오른쪽 캔버스에서 마우스를 드래그**하여 원하는 영역을 지정합니다.
   - 드래그 중 **초록색 점선 사각형**이 표시됩니다.
3. 마우스를 놓으면 ROI 좌표가 자동으로 저장됩니다.
4. **처음 설정하는 경우**, 드래그 영역의 이미지가 자동으로 템플릿으로 저장됩니다.
5. **이미 템플릿이 있는 경우**, ROI 좌표만 업데이트되고 기존 템플릿은 유지됩니다.

```
[드래그 동작 흐름]

  캔버스에서 드래그 시작 (마우스 버튼 누름)
       ↓
  초록색 점선 사각형 실시간 표시 (드래그 중)
       ↓
  마우스 버튼 놓기
       ↓
  ┌─── 캔버스 좌표를 실제 이미지 좌표로 변환 ───┐
  │  real_x = canvas_x / scale_factor           │
  │  real_y = canvas_y / scale_factor           │
  │  real_w = (x2 - x1) / scale_factor         │
  │  real_h = (y2 - y1) / scale_factor         │
  └──────────────────────────────────────────────┘
       ↓
  config["roi"][선택한_키] = {x, y, w, h} 저장
       ↓
  ┌── 템플릿 이미지가 없는 경우 ──┐
  │  스크린샷에서 해당 영역 잘라내기  │
  │  images/ 폴더에 PNG로 저장     │
  └──────────────────────────────┘
       ↓
  config.json에 자동 저장
```

### 5.2 설정 결과 확인

- **왼쪽 패널 "Selected ROI"** 섹션에 X, Y, W, H 좌표가 표시됩니다.
- **목록의 상태 태그**가 업데이트됩니다 (예: `[---]` → `[OK]`).
- **캔버스에 ROI 사각형**이 색상으로 표시됩니다.

## 6. 템플릿 이미지 등록하기

템플릿 이미지를 등록하거나 교체하는 방법은 4가지입니다.

### 6.1 자동 등록 (드래그 시 첫 번째)

ROI를 처음 드래그로 설정할 때, **템플릿이 아직 없는 경우** 자동으로 캡쳐 이미지에서 해당 영역을 잘라내어 저장합니다.

- 저장 위치: `images/{이미지키}.png`
- 상태바에 `+ template saved` 메시지가 표시됩니다.

### 6.2 스크린샷에서 교체 (Replace from Screenshot)

이미 저장된 스크린샷에서 현재 ROI 영역을 잘라내어 템플릿을 교체합니다.

1. ROI 항목을 선택합니다.
2. **"Replace from Screenshot"** 버튼을 클릭합니다.
3. 현재 캡쳐된 스크린샷에서 ROI 좌표에 해당하는 영역이 잘려서 저장됩니다.

> **주의**: 먼저 "Capture Screen"으로 스크린샷을 촬영해야 합니다.

### 6.3 실시간 화면에서 교체 (Replace from Live Screen)

타겟 윈도우의 **현재 실시간 화면**에서 직접 캡쳐합니다.

1. ROI 항목을 선택합니다.
2. **"Replace from Live Screen"** 버튼을 클릭합니다.
3. 에디터가 잠시 숨겨진 후, 타겟 윈도우의 해당 ROI 영역이 실시간으로 캡쳐됩니다.
4. 캡쳐된 이미지가 템플릿으로 저장됩니다.

> **활용**: 게임 화면이 변경된 경우, ROI 좌표는 유지하면서 템플릿 이미지만 업데이트할 때 유용합니다.

### 6.4 파일에서 불러오기 (Load from File...)

외부 이미지 파일을 직접 불러옵니다.

1. ROI 항목을 선택합니다.
2. **"Load from File..."** 버튼을 클릭합니다.
3. 파일 선택 대화상자에서 PNG/JPG/BMP 이미지를 선택합니다.
4. 선택한 이미지가 `images/` 폴더에 복사되어 템플릿으로 등록됩니다.

### 6.5 템플릿 미리보기

- 왼쪽 패널 하단의 **Template Image** 섹션에 현재 등록된 템플릿의 미리보기가 표시됩니다.
- 이미지 크기 정보도 함께 표시됩니다.

---

## 7. 수동으로 좌표 입력하기

드래그 대신 정확한 좌표를 직접 입력할 수 있습니다.

1. ROI 항목을 선택합니다.
2. **"Manual Entry"** 섹션의 X, Y, W, H 필드에 값을 입력합니다.
3. **"Apply Manual"** 버튼을 클릭합니다.
4. 좌표가 config에 저장되고 캔버스에 반영됩니다.

> **참고**: 수동 입력은 좌표만 설정하며, 템플릿 이미지는 별도로 등록해야 합니다.

---

## 8. ROI 항목 목록 및 상태 표시

왼쪽 목록에 표시되는 상태 태그의 의미:

| 태그 | 의미 |
|------|------|
| `[OK]` | ROI 좌표와 템플릿 이미지가 모두 설정됨 (준비 완료) |
| `[ROI]` | ROI 좌표만 설정됨, 템플릿 이미지 없음 |
| `[IMG]` | 템플릿 이미지만 있음, ROI 좌표 없음 |
| `[SET]` | 좌표만 필요한 항목 (템플릿 불필요)이 설정됨 |
| `[---]` | 아무것도 설정되지 않음 |

### ROI 항목 전체 목록

| ROI 키 | 설명 | 템플릿 필요 여부 |
|---------|------|:-:|
| `empty_slot` | 캐릭터 선택 화면의 빈 슬롯 | O |
| `knight_icon` | 클래스 선택 화면의 기사 아이콘 | O |
| `knight_verify` | 기사 이미지 확인 영역 | O |
| `name_input` | 캐릭터 이름 입력 필드 | X |
| `confirm_button` | 캐릭터 생성 확인 버튼 | O |
| `character_slot` | 캐릭터 선택 화면 슬롯 | X |
| `tab_area` | 탭 UI 영역 | X |
| `item_slot` | 인벤토리 아이템 슬롯 | O |
| `popup_text` | 팝업 텍스트 영역 | O |
| `scarecrow_search` | 허수아비 검색 영역 | O |
| `level_display` | 레벨 표시 영역 (OCR) | O |
| `mp_display` | MP 표시 영역 (OCR) | O |
| `exp_display` | 경험치 표시 영역 (멈춤 감지) | X |
| `hp_display` | HP 바 영역 (사망 감지) | X |
| `exit_button` | 나가기 버튼 영역 | O |
| `delete_button` | 삭제 버튼 영역 | O |

## 9. 허수아비(Scarecrow) 템플릿 설정

허수아비는 여러 방향의 이미지를 등록하여 다중 템플릿 매칭을 수행합니다.

### 9.1 Scarecrow Detection Editor 열기

- **메뉴바**: `Tools` → `Scarecrow Detection Editor`
- **설정 탭**: `Scarecrow Detection Editor` 버튼 클릭

### 9.2 다방향 템플릿 등록

1. **"Add from File"**: 외부 이미지 파일을 불러옵니다.
2. **"Capture from Screen"**: 타겟 윈도우를 캡쳐하고 드래그로 영역을 선택합니다.
3. **"Capture from ROI"**: `scarecrow_search` ROI 영역을 캡쳐한 후, 그 안에서 추가로 영역을 선택합니다.

각 템플릿은 `images/scarecrow_dir1.png`, `images/scarecrow_dir2.png` 등으로 저장되며, `config["scarecrow_templates"]` 리스트에 등록됩니다.

### 9.3 HSV 색상 필터 설정

허수아비 감지의 정확도를 높이기 위한 색상 필터링을 설정할 수 있습니다:

1. **"Auto-detect from ROI"**: `scarecrow_search` 영역의 중앙값 HSV를 자동으로 샘플링합니다.
2. **H/S/V 슬라이더**: 최소/최대 HSV 범위를 수동으로 조절합니다.
3. **"Preview Filter"**: HSV 마스크 미리보기를 확인합니다.
4. **"Test Detection"**: `find_scarecrow()` 함수를 실행하여 감지 결과를 테스트합니다.

## 10. 클릭 위치 설정

단일 좌표(x, y) 클릭 위치를 설정합니다.

### Click Position Editor 열기

- **메뉴바**: `Tools` → `Click Position Editor`
- **설정 탭**: `Open Click Position Editor` 버튼 클릭

### 사용 방법

1. 설정할 항목 옆의 **"Pick"** 버튼을 클릭합니다.
2. 타겟 윈도우 캡쳐 화면이 나타납니다.
3. 원하는 위치를 **한 번 클릭**합니다.
4. 선택한 좌표가 윈도우 기준 상대 좌표로 저장됩니다.

## 11. 좌표 체계 설명

### 윈도우 상대 좌표 (Window-Relative)

모든 ROI 및 클릭 위치 좌표는 **타겟 윈도우의 좌상단(0, 0) 기준**으로 저장됩니다.

```
타겟 윈도우 (화면 위치: 100, 200)
┌───────────────────────────┐
│ (0, 0)                    │  ← 윈도우 좌상단 = 원점
│                           │
│    ┌────────┐             │
│    │  ROI   │ (50, 80)    │  ← config에 저장되는 좌표
│    │ 100x60 │             │
│    └────────┘             │
│                           │
└───────────────────────────┘
```

### 런타임 절대 좌표 변환

자동화 실행 시, 윈도우 오프셋을 더하여 절대 화면 좌표로 변환됩니다:

```
절대 좌표 = ROI 좌표 + 윈도우 위치

abs_x = roi["x"] + window_offset_x   →  50 + 100 = 150
abs_y = roi["y"] + window_offset_y   →  80 + 200 = 280
```

> **장점**: 윈도우가 이동하더라도 ROI 좌표를 다시 설정할 필요가 없습니다.

## 12. 설정 저장 및 적용

### 자동 저장

에디터에서 수행하는 모든 변경사항(ROI 드래그, 수동 입력, 템플릿 교체 등)은 **config.json에 자동으로 저장**됩니다.

### 저장되는 구조

```json
{
  "roi": {
    "empty_slot": {"x": 100, "y": 200, "w": 80, "h": 60},
    "knight_icon": {"x": 150, "y": 300, "w": 50, "h": 50},
    ...
  },
  "images": {
    "empty_slot": "images/empty_slot.png",
    "knight_icon": "images/knight_icon.png",
    "item_icon": "images/item_icon.png",
    ...
  },
  "scarecrow_templates": [
    "images/scarecrow_dir1.png",
    "images/scarecrow_dir2.png"
  ]
}
```

### 파일 구조

```
PointNGD/
├── config.json              ← ROI 좌표 및 이미지 경로 저장
├── images/                  ← 템플릿 이미지 저장 폴더
│   ├── empty_slot.png
│   ├── knight_icon.png
│   ├── item_icon.png
│   ├── scarecrow_dir1.png
│   ├── scarecrow_dir2.png
│   └── ...
└── gui/
    ├── roi_editor.py        ← ROI & Image Editor 코드
    ├── scarecrow_editor.py  ← Scarecrow Detection Editor 코드
    └── window_utils.py      ← 윈도우 캡쳐 유틸리티
```

## 전체 설정 워크플로우 요약

```
1. 타겟 프로그램 실행 및 원하는 화면 준비
        ↓
2. Settings에서 Target Window Title 설정
        ↓
3. ROI & Image Editor 열기
        ↓
4. "Capture Screen" 클릭 → 타겟 윈도우 캡쳐
        ↓
5. 목록에서 ROI 항목 선택
        ↓
6. 캔버스에서 드래그하여 영역 지정
   → ROI 좌표 저장 + 템플릿 이미지 자동 등록
        ↓
7. 모든 ROI 항목에 대해 5~6 반복
        ↓
8. (선택) Scarecrow Detection Editor에서 다방향 템플릿 등록
        ↓
9. (선택) Click Position Editor에서 클릭 위치 설정
        ↓
10. 모든 항목이 [OK] 또는 [SET] 상태인지 확인
        ↓
11. 자동화 시작!
```
